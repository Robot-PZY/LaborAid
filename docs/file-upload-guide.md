# 劳权智助 · LaborAid 文件上传与多模态说明

LaborAid 支持在网页中直接上传文件，系统会提取文字后交给 AI 分析。**并非所有格式都具备完整多模态能力**，请按场景选择页面和模型。

---

## 各页面文件上传能力

| 功能 | 页面 | 网页上传 | 拖拽上传 | 支持格式 |
|------|------|:--------:|:--------:|----------|
| **快速咨询（首页）** | `/` | ✅ | ❌ | 图片、PDF、Word（最多 3 个） |
| **整理证据** | `/evidence` | ✅ | ✅ | PDF、Word、Excel、TXT、图片、音频 |
| **审查合同** | `/contracts` | ✅ | ✅ | PDF、Word、Excel、TXT、图片 |
| **生成文书** | `/documents` | ✅ | ❌ | 同上 + 音频（提取后填入案情） |
| **分析案情** | `/research` | ✅ | ❌ | 同上 + 音频 |
| **法律知识库** | `/knowledge` | ✅ | ✅ | PDF、Word、Excel、TXT、图片 |
| **检索法规** | `/search` | ❌ | ❌ | 仅文字检索 |
| **管理案件** | `/cases` | ❌ | ❌ | 无直接上传 |

### 支持的扩展名

```
.pdf .docx .doc .txt .xlsx .xls
.png .jpg .jpeg .gif .webp .bmp .tiff
.mp3 .wav .m4a .ogg .flac .aac .wma  （部分页面）
```

默认单文件大小上限：**50 MB**（可在 Settings → 系统配置 中调整）。

## 后端如何处理不同文件

```
上传文件
  │
  ├─ PDF（有文字层）     → pypdf 本地提取 ✅
  ├─ PDF（扫描件）       → 转图片 + LLM 视觉 OCR ⚠️ 需视觉模型
  ├─ Word / Excel / TXT  → 本地库解析 ✅
  ├─ 图片               → 百炼 OCR ⚠️ 需配置 VISION_LLM_*（qwen-vl-ocr-latest）
  └─ 音频               → Whisper API ⚠️ 国内 API 通常不支持
```

## 能力评估

| 能力 | 评级 | 说明 |
|------|:----:|------|
| 文档上传（PDF/Word/Excel） | ⭐⭐⭐⭐⭐ | 合同、文书、知识库均支持 |
| 图片 OCR（证据照片） | ⭐⭐⭐ | 需配置视觉模型，见 [模型配置指南](./model-config-guide.md) |
| 扫描 PDF OCR | ⭐⭐⭐ | 合同审查模块已实现，依赖视觉模型 |
| 音频转写 | ⭐ | 仅 Whisper 兼容 API；否则提示手动转写 |
| 视频 | ❌ | 不支持 |
| 批量多文件 | ⭐⭐ | 证据需逐条关联上传 |

## 典型使用场景

### ✅ 已支持良好

- 上传 Word/PDF 合同 → **审查合同** 审查
- 上传起诉状 PDF → **生成文书** 「上传文件提取文字」填入案情
- 上传证据图片 → **整理证据** OCR + AI 分析
- 上传法规 PDF → **法律知识库** 入库检索

### ⚠️ 需注意

- 只配置了 DeepSeek 文本模型，却上传扫描件/照片 → OCR 会失败
- 上传庭审录音 → 需 Whisper 或第三方语音服务（阿里云/讯飞等）
- 在 **检索法规** 检索页无法直接传文件搜法条
