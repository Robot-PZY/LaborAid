# LaborAid 快速启动（本地开发）

## 最省事：双击启动

在仓库根目录（与 `README.md` 同级）：


| 操作     | 文件                                                 |
| ------ | -------------------------------------------------- |
| **启动** | 双击 `**start-laboraid.bat`**                        |
| **停止** | 双击 `**stop-laboraid.bat`**（或关掉弹出的两个 PowerShell 窗口） |


> 不要用中文文件名的 `.bat`。项目路径里常有空格（如 `Skill Learning`），中文 bat 还容易乱码，导致 `'Skill' 不是内部或外部命令` 这类报错。

启动后会自动：

1. 清理本项目的 **5320**（前端）、**8010**（后端）端口
2. 打开 **Backend**、**Frontend** 两个 PowerShell 窗口（不要关）
3. 浏览器打开登录页

### 访问地址


| 用途     | 地址                                                                                   |
| ------ | ------------------------------------------------------------------------------------ |
| 用户端    | [http://127.0.0.1:5320](http://127.0.0.1:5320)                                       |
| 管理端    | [http://127.0.0.1:5320/login?portal=admin](http://127.0.0.1:5320/login?portal=admin) |
| API 文档 | [http://127.0.0.1:8010/docs](http://127.0.0.1:8010/docs)                             |


**默认管理员**：`Admin` / `123456`（或 `admin@laboraid.local` / `123456`）

## 在 Cursor / VS Code 里启动

1. `Ctrl+Shift+P` → **Tasks: Run Task**
2. 选 **LaborAid: 启动开发环境**

停止：同样方式运行 **LaborAid: 停止开发环境**。

---

## 命令行启动（可选）

```powershell
cd "F:\Undergraduate\Skill Learning\LaborAid"
.\scripts\dev.ps1
```

路径有空格时请保留引号。

---

## 首次使用前要装什么？

- **Python 3.11+**（后端）  
- **Node.js 18+**（前端）  
- 后端依赖：`cd LaborAid\backend` → `pip install -r requirements.txt`  
- 前端依赖：`cd LaborAid\frontend` → `npm install`

后端配置：复制 `LaborAid\backend\.env.example` 为 `.env`（可选）。

---

## 启动失败时看一眼


| 现象                  | 处理                                                       |
| ------------------- | -------------------------------------------------------- |
| `'Skill' 不是内部或外部命令` | 已修复：请用 `**start-laboraid.bat`**，并更新到最新 `scripts/dev.ps1` |
| 登录一直转圈              | 看 **Backend** 窗口有没有报错                                    |
| 前端打不开               | 看 **Frontend** 窗口；或先运行 **stop-laboraid.bat** 再启动         |
