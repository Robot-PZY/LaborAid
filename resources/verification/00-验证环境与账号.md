# 00 · 验证环境与账号

## 1. 环境要求

| 项目 | 要求 |
|------|------|
| Python | 3.12+ |
| Node.js | 18+ |
| 浏览器 | Chrome / Edge（推荐，支持 Web Speech 语音输入） |
| 网络 | 可访问所配置的 LLM API |

## 2. 启动命令（首次建议完整执行）

```powershell
# 终端 1：后端
cd LaborAid/backend
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt
python -m uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

# 终端 2：前端
cd LaborAid/frontend
npm install
npm run dev
```

| 服务 | 地址 |
|------|------|
| 用户端 | http://localhost:5173/ |
| 管理端 | http://localhost:5173/login?portal=admin |
| API 文档 | http://localhost:8000/docs |

## 3. 测试账号

### 3.1 管理员（预置，首次启动自动创建）

| 项目 | 值 |
|------|------|
| 登录名 | `Admin` 或 `admin@laboraid.local` |
| 密码 | `123456` |
| 入口 | `/login?portal=admin` |

### 3.2 普通用户（验证注册功能时临时创建）

| 项目 | 建议值 |
|------|--------|
| 姓名 | `验证用户张三` |
| 邮箱 | `verify.demo@example.com`（须为合法域名，勿用 `.local`） |
| 密码 | `DemoPass123`（≥8 位，含字母与数字） |

> 验证完成后可在管理端禁用或删除测试用户，保持环境整洁。

## 4. AI 能力配置检查

在管理端 **模型配置** 确认以下项（无 Key 时部分功能无法完整验证）：

| 能力 | 环境变量 | 影响模块 |
|------|----------|----------|
| 文本 LLM | `LLM_API_KEY`、`LLM_MODEL` | 维权前台、生成文书、分析案情、审查合同、检索法规摘要 |
| 视觉 OCR | `VISION_LLM_API_KEY` | 整理证据图片识别、维权前台图片上传 |
| 向量检索 | Chroma 本地 / 远程 | 检索法规、知识库检索 |

## 5. 启动自检清单（必做）

| 序号 | 检查项 | 预期 | 结果 |
|------|--------|------|------|
| 1 | 访问用户端首页 | 页面正常加载 | ☐ |
| 2 | 管理端登录 | 进入 `/admin` | ☐ |
| 3 | 用户端注册 + 登录 | 进入首页 Dashboard | ☐ |
| 4 | 侧边栏各模块可打开 | 无 404 / 白屏 | ☐ |

## 6. 常见问题

| 现象 | 处理 |
|------|------|
| 注册报 422 | 邮箱域名不合法，改用 `@qq.com`、`@example.com` 等 |
| AI 功能超时 | 检查 `.env` 中 API Key 与网络 |
| 图片 OCR 失败 | 确认已配置 `VISION_LLM_*` |
| 前端端口非 5173 | 旧进程占用，关闭后重启 `npm run dev` |
