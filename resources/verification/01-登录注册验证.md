# 01 · 登录注册验证

## 验证目标

确认用户端注册、登录、登出及管理端登录鉴权正常，错误输入能被正确拦截。

> 前置条件：已完成 [00-验证环境与账号](./00-验证环境与账号.md)。

---

## A. 用户端注册

**路径**：http://localhost:5173/login → 切换到「注册」

| 序号 | 操作 | 输入数据 | 预期结果 | 结果 |
|------|------|----------|----------|------|
| A1 | 正常注册 | 姓名：`验证用户张三`<br>邮箱：`verify.demo@example.com`<br>密码：`DemoPass123`<br>确认密码：同上 | 注册成功，提示可登录 | ☐ |
| A2 | 重复邮箱 | 再次提交 A1 相同邮箱 | 提示「该邮箱已注册」 | ☐ |
| A3 | 弱密码 | 邮箱：`weak@example.com`<br>密码：`12345678` | 提示需包含字母 | ☐ |
| A4 | 密码过短 | 密码：`abc12` | 提示长度不足 | ☐ |
| A5 | 邮箱格式错误 | 邮箱：`not-an-email` | 前端或后端校验失败 | ☐ |

---

## B. 用户端登录

**路径**：http://localhost:5173/login

| 序号 | 操作 | 输入 | 预期结果 | 结果 |
|------|------|------|----------|------|
| B1 | 正确登录 | `verify.demo@example.com` / `DemoPass123` | 跳转首页，显示用户名 | ☐ |
| B2 | 错误密码 | 正确邮箱 + 错误密码 | 提示账号或密码错误 | ☐ |
| B3 | 记住邮箱 | 勾选「记住邮箱」后退出再进 | 邮箱自动填充 | ☐ |
| B4 | 登出 | 侧边栏退出登录 | 回到登录页，需重新登录 | ☐ |

---

## C. 管理端登录

**路径**：http://localhost:5173/login?portal=admin

| 序号 | 操作 | 输入 | 预期结果 | 结果 |
|------|------|------|----------|------|
| C1 | 账号名登录 | `Admin` / `123456` | 进入管理端概览 | ☐ |
| C2 | 邮箱登录 | `admin@laboraid.local` / `123456` | 同上 | ☐ |
| C3 | 普通用户访问管理端 | 用 B1 账号登录后手动访问 `/admin` | 被拒绝或重定向 | ☐ |
| C4 | 管理端无注册入口 | 查看管理端登录页 | 不显示用户注册 | ☐ |

---

## D. 接口级快速验证（可选）

使用 API 文档 http://localhost:8000/docs 或命令行：

```http
POST /api/v1/auth/register
Content-Type: application/json

{
  "name": "API测试用户",
  "email": "api.test@example.com",
  "password": "ApiTest123"
}
```

```http
POST /api/v1/auth/login
Content-Type: application/json

{
  "email": "api.test@example.com",
  "password": "ApiTest123"
}
```

登录成功后携带 Token 调用：

```http
GET /api/v1/auth/me
Authorization: Bearer {access_token}
```

| 序号 | 接口 | 预期 | 结果 |
|------|------|------|------|
| D1 | POST register | 201，返回用户信息 | ☐ |
| D2 | POST login | 200，返回 access_token | ☐ |
| D3 | GET /me | 200，role 为 lawyer | ☐ |
